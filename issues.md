# Issues

## 1. 踩过的坑

### 1.1 占总股本比例和占流通股比例混淆

在抓取股东减持数据时，一开始把网站返回的“占总股本比例”和“占流通股比例”混淆了，导致筛选结果和最终表格中的比例字段出现问题。后续检查字段含义后，统一使用“占总股本比例”作为最终结果字段，并根据该字段参与后续计算和输出。

### 1.2 公告标题关键词不统一

在提取公告时发现，不同公司的公告标题没有统一规范。有的公告标题中包含“减持”，有的包含“股东权益”，有的包含“权益变动”。一开始只使用单一关键词进行匹配，导致很多相关公告没有被抓取到。后续将关键词扩展为“减持”“股东权益”“股东”“权益变动”等，提高公告匹配覆盖率。

### 1.3 东方财富返回结果不是标准 JSON

抓取东方财富接口数据时，发现部分接口返回的不是标准 JSON，而是 JSONP 格式，不能直接用 `json.loads()` 解析。因此需要先对返回文本进行处理，去掉外层回调函数，再转换为 JSON 数据。

### 1.4 单页最多只能返回 100 条数据

东方财富公告接口限制单个 page 最多返回 100 条数据。因此即使将 `page_size` 设置为 200，一次请求也只能读取 100 条数据。为了解决这个问题，需要通过更换 `page_index` 的方式读取不同页的数据，例如分别请求 `page_index=1` 和 `page_index=2`。

### 1.5 误把 EITIME 当成减持结束日

一开始把字段 `EITIME` 当成减持结束日使用，但后来发现真正的减持结束日字段是 `END_DATE`，而 `EITIME` 是东方财富数据库中的记录更新时间。后续改为使用 `END_DATE` 作为减持结束日。

### 1.6 请求超时时间设置过短

一开始设置的 runtime 或 timeout 过小，导致部分股票的数据还没来得及返回，请求就已经超时失败。后续增加了请求超时时间，并加入重试机制，降低因为网络波动或接口响应慢导致的抓取失败。

### 1.7 公告原文不能直接从网页抓取

一开始以为公告是普通网页，可以直接抓取正文内容。但实际处理时发现，公告正文通常以 PDF 附件形式存在，不能直接从网页 HTML 中提取完整内容。因此需要先通过公告接口找到对应 PDF 链接，再使用 PDF 解析工具解码公告内容。

### 1.8 减持公告当天可能存在多条相近公告

部分公司在同一天会发布多条标题相近的公告，例如都包含“减持”“权益变动”等关键词。一开始只取其中一条公告，容易出现抓取错误或遗漏。最终改为在同一天匹配到多条减持相关公告时，将多个公告链接全部保存，避免误删有效公告。

## 2. 未解决的问题

### 2.1 部分公司缺少预披露公告或减持原因

一部分公司没有发布预披露公告，或者虽然发布了减持公告，但公告中没有明确披露减持原因。因此这类公司无法通过当前规则抓取到对应的减持原因字段。

### 2.2 多条相似公告难以自动判断目标公告

部分公司在减持公告当天发布多条含有类似表述的公告。由于公告内容必须先获取 PDF 链接并完成 PDF 转码后才能得到正文，因此在公告列表阶段无法直接判断哪一条才是最终需要的公告。对于这种情况，当前处理方式是将多个链接都保存下来，如有需要再人工筛选。

### 2.3 减持原因和股东类型提取规则无法完全通用

当前减持原因主要从预披露公告中抓取，股东类型主要从减持公告中抓取。但是不同公司公告的表述方式差异较大，没有一套完全通用的正则规则。此外，部分公司减持公告中没有明确提及股东类型，因此该字段也可能无法成功提取。

### 2.4 数据库数据与公告原文存在不一致

极少部分公司存在东方财富数据库减持数据与公告原文不一致的情况。例如公告中写明减持期持续三个月，但数据库中记录的减持期间只有十多天。为了保持整体流程统一，最终优先采用数据库中的结构化数据。

### 2.5 扫描件 PDF 无法直接解析

个别公司的公告 PDF 可能是扫描件。这类文件无法通过普通 PDF 文本解析工具直接提取文字，需要引入 OCR 工具识别图片中的文字。当前代码暂未加入 OCR，因此扫描件公告的文本信息可能无法成功解析。

## 3. 后续改进方向

### 3.1 扩大关键词词库

后续可以继续扩大关键词词库，用于优化减持原因、股东身份、公告类型等文字信息的抓取效果。例如针对不同公司常见的标题写法和正文表达方式，持续补充关键词和匹配规则。不过这一部分需要不断更新，因为不同公司公告表述方式并不完全一致。

### 3.2 进一步解析预披露公告内容

如果关键词积累足够多，可以进一步从最新的预披露公告中抓取预计减持数量、预计减持比例和预计减持期间等信息。但该功能需要反复优化，因为不同公司文本表述差异较大，一旦大部分信息都依赖文本提取，就容易出现字段识别错误或截取错误。

### 3.3 增加更多正则规则

目前在提取减持原因等文本字段时，仍然存在部分文本没有及时截断的问题。后续可以加入更多正则表达式规则，例如根据句号、分号、表格字段、换行位置、常见公告小标题等进行多层判断，提高文本截取的准确性。

### 3.4 加入 OCR 处理扫描件公告

对于扫描版 PDF，可以后续加入 OCR 工具进行文字识别。这样可以提高公告文本解析的覆盖率，减少因为 PDF 无法直接解码而导致的字段缺失。

### 3.5 增加人工复核标记

对于多条公告链接、字段缺失、PDF 解析失败、减持原因为空等情况，可以在结果表中增加“是否需要人工复核”字段。这样可以将自动化结果和人工检查结合起来，提高最终数据表的可靠性。

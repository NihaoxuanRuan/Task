# README.md

## 1. 项目结构

```text
Task/
│
├── .git/                 # Git 版本管理目录，保留完整 commit 历史
│
├── script.py             # 主程序：获取股票池、抓取减持数据、匹配公告、解析 PDF
├── result.xlsx           # 最终结构化结果表
├── README.md             # 项目说明文档，包括项目结构、运行方式、依赖和数据源选型理由
├── report.md             # 数据画像报告
├── issues.md             # 踩过的坑、未解决的问题和改进方向
└── ai_usage.md           # AI 工具使用说明
```

其中，`script.py` 是项目核心代码，主要完成以下步骤：

```text
1. 使用 AkShare 获取中证1000成分股列表；
2. 遍历股票池，抓取最近三个月的股东减持记录；
3. 根据减持前持股比例筛选 5% 以上股东减持数据；
4. 根据股票代码和公告日期匹配东方财富公告；
5. 获取减持公告和预披露公告的 PDF 链接；
6. 解析公告 PDF 原文；
7. 提取股东类型、减持原因、公告原文链接等字段；
8. 输出最终结构化数据表 result.xlsx。
```

`result.xlsx` 为最终交付的数据表，字段包括：

```text
股票代码 / 简称 / 公告日期 / 减持股东 / 股东类型 / 减持股数 / 占总股本比例 / 减持期间 / 减持原因 / 减持公告链接 / 预披露公告链接
```

## 2. 如何运行

首先安装依赖：

```bash
pip install -r requirements.txt
```

然后运行主程序：

```bash
python main.py
```

程序会自动完成以下流程：

```text
1. 获取中证1000成分股列表
2. 遍历每只股票并抓取减持记录
3. 按公告日期和5%以上股东条件筛选数据
4. 根据股票代码获取公告列表
5. 匹配减持公告和预披露公告
6. 获取公告 PDF 链接
7. 解析 PDF 原文并提取减持原因、股东类型
8. 输出 result_df_enriched
```

如果需要保存 Excel，可以在代码最后取消注释

## 3. Requirements

`requirements.txt` 内容如下：


各依赖作用：

```text
akshare      获取中证1000成分股

curl-cffi    模拟浏览器请求 PDF，降低请求失败概率
openpyxl     保存 Excel 文件
```

## 4. 数据源选型理由

### 4.1 中证1000成分股数据源：AkShare

选择 AkShare 的原因是它可以直接获取中证1000指数成分股，返回结果为 DataFrame，便于后续用 pandas 处理。同时，使用接口获取股票池可以避免手动下载和维护成分股名单。

### 4.2 股东减持数据源：东方财富数据中心

股东减持记录来自东方财富数据中心接口。该接口提供结构化字段，包括股票代码、股票简称、公告日期、减持股东、减持股数、减持比例、减持后持股比例、减持开始日和减持结束日。相比直接解析公告 PDF，结构化接口更适合作为初始数据来源，方便批量筛选和计算减持前持股比例。同时不选择巨潮资讯网的原因是巨潮资讯网的数据只含有日期、股东名称、减持数量和比例以及交易价格，缺少股东总持股比例等关键信息，因此选择东方财富的数据。

### 4.3 公告链接数据源：东方财富公告接口

减持公告和预披露公告来自东方财富公告接口。该接口可以根据股票代码查询公告列表，并通过 `art_code` 获取公告 PDF 链接。使用公告接口的原因是它可以将结构化减持记录与原始公告对应起来，便于补充减持公告链接、预披露链接以及后续原文核验。

### 4.4 PDF 原文解析工具：PyMuPDF

公告 PDF 使用 PyMuPDF 解析。选择它的原因是 PyMuPDF 可以直接从 PDF 二进制内容中提取文本，不需要手动下载到本地文件，适合批量处理公告原文，并进一步用正则表达式提取减持原因和股东类型。
